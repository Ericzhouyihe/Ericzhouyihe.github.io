用决策树读懂鸢尾花分类：从基尼系数到模型评估

一、要解决什么问题

决策树是一种非常直观的机器学习算法。它的思路很像我们日常做判断：

如果花瓣长度很短，那么它大概率是山鸢尾；否则继续看花瓣宽度，再继续细分。

也就是说，决策树会把一个复杂问题拆成一连串“是 / 否”判断，最后走到某个叶子节点，得到分类结果。

本文基于鸢尾花数据集，完整梳理一个使用 sklearn.tree.DecisionTreeClassifier 完成分类预测的流程，包括：

  4. 决策树模型训练；
  5. 准确率、精确率、召回率、F1 值评估；
  6. ROC 和 AUC 理解；
  7. 决策树可视化结果解读。

二、鸢尾花数据集是什么

鸢尾花数据集是机器学习入门中非常经典的数据集。它的任务是：根据一朵花的 4 个测量值，预测它属于哪一种鸢尾花。

数据中的 4 个特征通常表示为：

特征 含义
A 或 sepal length 花萼长度
B 或 sepal width 花萼宽度
C 或 petal length 花瓣长度
D 或 petal width 花瓣宽度

目标类别有 3 种：

原始类别 编码后
Iris-setosa 0
Iris-versicolor 1
Iris-virginica 2

可以把这个问题想象成一个植物学小助手：你告诉它花萼和花瓣的尺寸，它根据历史样本总结出来的规则，判断这朵花最像哪一种。

九、决策树的核心思想

决策树每次都会选择一个特征和一个阈值，把数据分成两部分，让分裂后的数据尽量“纯”。

所谓“纯”，可以理解为：

一个节点里最好都是同一种类别。

比如一个盒子里全是苹果，那么它很纯；
如果里面既有苹果、梨、香蕉，就不纯。

决策树常用的不纯度指标有基尼系数和信息熵。

十、基尼系数原理

代码中使用的模型默认评价指标是 gini，也就是基尼系数。

基尼系数公式为：

Gini(D)=1−∑k=1Kpk2Gini(D) = 1 - \sum_{k=1}^{K} p_k^2Gini(D)=1−∑k=1K​pk2​

其中：

  • DDD 表示当前数据集；
  • KKK 表示类别数量；
  • pkp_kpk​ 表示第 kkk 类样本在当前节点中的占比。

如果一个节点里所有样本都属于同一类，例如类别分布是：

1,0,01, 0, 01,0,0

则：

Gini(D)=1−(12+02+02)=0Gini(D) = 1 - (1^2 + 0^2 + 0^2) = 0Gini(D)=1−(12+02+02)=0

基尼系数为 000，表示节点非常纯。

如果一个节点里三类样本比较混杂，比如：

13,13,13\frac{1}{3}, \frac{1}{3}, \frac{1}{3}31​,31​,31​

则：

Gini(D)=1−((13)2+(13)2+(13)2)=23Gini(D) = 1 - ((\frac{1}{3})^2 + (\frac{1}{3})^2 + (\frac{1}{3})^2) = \frac{2}{3}Gini(D)=1−((31​)2+(31​)2+(31​)2)=32​

基尼系数越大，说明越混乱；基尼系数越小，说明越纯。

十一、信息熵补充理解

虽然本模型使用的是基尼系数，但决策树中也经常使用信息熵。

信息熵公式为：

H(D)=−∑k=1Kpklog⁡2pkH(D) = - \sum_{k=1}^{K} p_k \log_2 p_kH(D)=−∑k=1K​pk​log2​pk​

信息熵也是衡量混乱程度的指标：

  • 熵越大，数据越混乱；
  • 熵越小，数据越确定；
  • 如果所有样本都属于同一类，熵为 000。

生活化理解：
如果你闭眼从袋子里摸糖，袋子里全是苹果味糖果，你不用猜也知道摸到什么，信息熵低。
如果袋子里有很多种口味，而且比例接近，你就很难猜，信息熵高。

max_depth=3 限制树的最大深度为 3

为什么要限制深度？

如果不限制，决策树可能会把训练数据分得特别细，甚至记住每一个训练样本。这样训练集表现很好，但测试集可能表现变差，这种现象叫过拟合。

可以把它理解成：
一个学生把练习册答案全背下来了，练习题全对，但换一套新题就不会了。

限制树深度，就是让模型不要学得太死。

常见参数还有：

参数 作用
criterion 不纯度指标，可选 gini 或 entropy
splitter 特征划分方式，可选 best 或 random
max_depth 最大树深度
min_samples_split 节点至少有多少样本才允许继续分裂
min_samples_leaf 叶子节点至少保留多少样本
class_weight 类别权重，适合类别不平衡问题

几个指标的含义如下。

1. 准确率 Accuracy

准确率表示所有预测中，有多少预测对了：

Accuracy=预测正确的样本数总样本数Accuracy = \frac{预测正确的样本数}{总样本数}Accuracy=总样本数预测正确的样本数​

本例中：

  • 训练集准确率为 0.98330.98330.9833；
  • 测试集准确率为 0.96670.96670.9667。

说明模型整体表现很好。

2. 精确率 Precision

精确率表示：模型预测为某一类的样本中，有多少是真的。

Precision=TPTP+FPPrecision = \frac{TP}{TP + FP}Precision=TP+FPTP​

其中：

  • TPTPTP 表示真正例；
  • FPFPFP 表示假正例。

生活化理解：
如果模型说“这些都是 Iris-versicolor”，精确率关心的是：这些被模型点名的样本里，有多少真的就是 Iris-versicolor。

3. 召回率 Recall

召回率表示：真实属于某一类的样本中，有多少被模型找出来了。

Recall=TPTP+FNRecall = \frac{TP}{TP + FN}Recall=TP+FNTP​

其中：

  • FNFNFN 表示假负例。

生活化理解：
如果真实有 100 个目标客户，模型找出了 90 个，那么召回率就是 9090%90。

4. F1-score

F1 值是精确率和召回率的综合指标：

F1=2×Precision×RecallPrecision+RecallF1 = \frac{2 \times Precision \times Recall}{Precision + Recall}F1=Precision+Recall2×Precision×Recall​

当我们既关心“找得准”，又关心“找得全”时，F1 值很有用。

二十一、为什么决策树适合入门

决策树的优点非常明显：

  1. 可解释性强：可以画出树图，看到每一步判断；
  2. 不需要复杂的数据预处理；
  3. 可以处理分类问题，也可以处理回归问题；
  4. 对特征尺度不敏感；
  5. 非线性能力较强，能学习复杂边界。

不过它也有缺点：

  1. 容易过拟合；
  2. 对数据扰动比较敏感；
  3. 单棵树的稳定性不如随机森林、梯度提升树；
  4. 如果树太深，规则会变得复杂，不容易泛化。

因此实际项目中，决策树常常作为理解模型原理的第一站。
如果追求更强效果，可以进一步学习：

  • 随机森林；
  • AdaBoost；
  • GBDT；
  • XGBoost；
  • LightGBM；
  • CatBoost。

这些算法本质上都和树模型有密切关系。
