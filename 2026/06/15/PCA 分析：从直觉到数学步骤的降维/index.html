PCA 主成分分析：从直觉到数学步骤的降维笔记

PCA 分析：从直觉到数学步骤的降维

1. PCA 是什么？

PCA，全称 Principal Component Analysis，中文叫 主成分分析。

它是一种经典的 无监督线性降维算法，主要作用是：

将高维数据映射到低维空间中，在尽可能保留原始信息的前提下，减少特征数量。

PCA 并不是简单地从原始特征中挑几个“重要特征”，然后丢掉其他特征。

它真正做的是：

把原始特征重新组合，生成一组新的特征，这些新特征叫做主成分。

这些主成分具有两个特点：

  1. 每个主成分都是原始特征的线性组合
  2. 主成分之间彼此正交，也就是线性无关

2. 为什么需要降维？

高维数据经常会带来很多问题。

比如一张 256 × 256 的灰度图，就有 65536 个像素特征。如果直接拿这些特征训练模型，可能会遇到以下问题：

  1. 计算成本高

    特征越多，模型训练和推理消耗的时间、内存越大。

  2. 容易过拟合

    当特征数量远大于样本数量时，模型可能学到噪声，而不是数据规律。

  3. 难以可视化

    人类只能直观理解二维或三维数据，高维数据很难直接观察。

  4. 存在冗余信息

    很多特征之间高度相关，携带的信息是重复的。

PCA 的目标就是：

用更少的维度，保留尽可能多的信息，同时减少特征之间的冗余。

3. PCA 的核心直觉

可以先从一个二维降到一维的例子理解 PCA。

假设有一批学生的两门成绩：

  • 数学成绩
  • 物理成绩

最笨的办法是只看数学成绩，直接丢掉物理成绩。

PCA 的做法是：

这个综合分，就是第一主成分，也就是 PC1。

4. 什么是“信息最多”？

在 PCA 中，通常用 方差 来衡量信息量。

简单来说：

  • 方差大：数据分布更分散，更容易区分样本
  • 方差小：数据挤在一起，区分度较低

所以 PCA 要找的第一个方向，就是：

让数据投影过去以后，方差最大的方向。

这个方向就是 第一主成分 PC1。

如果还需要第二个主成分，PCA 会继续寻找一个新的方向：

  1. 与 PC1 正交
  2. 在剩余方向中方差最大

这个方向就是 第二主成分 PC2。

5. 一个形象比喻：最佳拍照角度

可以把 PCA 想象成给一堆三维物体拍照。

真实世界是三维的，而照片是二维的。

拍照本质上就是一种降维：
把三维信息压缩到二维平面中。

但不同角度拍出来的照片，信息保留程度不同。

比如桌上有一堆笔和本子：

  • 从正上方拍，物体分布清楚
  • 从侧面拍，东西可能重叠，看不清楚

PCA 要做的事情，就是找到那个最能保留信息的“拍照角度”。

第一主成分就是最重要的拍摄方向。
第二主成分是在不重复第一方向的前提下，补充最多信息的方向。

6.1 数据标准化

PCA 对特征尺度非常敏感，所以通常需要先做标准化。

常用方法是 Z-Score 标准化：

[ x’ = \frac{x - \mu}{\sigma} ]

其中：

  • $\mu$ 是特征均值
  • $\sigma$ 是特征标准差

标准化后，每个特征的均值为 0，标准差为 1。

这一步很重要，因为如果不标准化，数值范围大的特征可能会主导主成分方向。

6.2 计算协方差矩阵

协方差用来衡量两个变量之间的线性相关程度。

标准化后，协方差矩阵可以写成：

[ C = \frac{1}{m-1}X^TX ]

协方差矩阵反映了不同特征之间的相关关系。

6.3 求特征值和特征向量

对协方差矩阵做特征分解：

[ Cv = \lambda v ]

可以得到：

  • 特征值 $\lambda$
  • 特征向量 $v$

在 PCA 中：

  • 特征值表示对应主成分的方差
  • 特征向量表示主成分的方向

特征值越大，说明该主成分保留的信息越多。

6.4 选择前 k 个主成分

将特征值从大到小排序：

[ \lambda_1 \geq \lambda_2 \geq … \geq \lambda_n ]

7. 如何选择 k 值？

k 是 PCA 中非常重要的参数，表示最终保留多少个主成分。

常见方法有两种。

7.1 累计方差贡献率

单个主成分的方差贡献率为：

[ \frac{\lambda_i}{\sum_{j=1}^{n}\lambda_j} ]

前 k 个主成分的累计方差贡献率为：

[ \frac{\sum_{j=1}^{k}\lambda_j}{\sum_{j=1}^{n}\lambda_j} ]

7.2 肘部法则

也可以画出“主成分序号 - 特征值”的折线图。

通常曲线一开始下降很快，后面逐渐变平。

那个明显变平的位置，就是所谓的“肘部”。

肘部之前的主成分贡献较大，肘部之后的主成分贡献较小，可以考虑丢弃。

8. PCA 的优点

PCA 的优点主要有：

  1. 不需要标签

    PCA 是无监督算法，可以直接用于没有标签的数据。

  2. 降低计算成本

    特征数量减少后，模型训练速度通常会变快。

  3. 缓解过拟合

    去掉部分低方差信息后，可以减少模型学习噪声的风险。

  4. 消除线性相关性

    主成分之间彼此正交，可以缓解多重共线性问题。

  5. 有一定去噪能力

    噪声往往出现在低方差方向上，丢弃这些方向可能有助于去噪。

9. PCA 的缺点

PCA 也有明显限制：

  1. 只能处理线性结构

    如果数据中存在复杂的非线性关系，普通 PCA 效果可能不好。

  2. 对异常值敏感

    异常值会影响协方差矩阵，从而改变主成分方向。

  3. 主成分解释性下降

    主成分是多个原始特征的线性组合，含义可能不如原始特征直观。

  4. 一定会损失信息

    只要进行了降维，就不可避免会丢失部分细节。

10. PCA 的常见变体

变体 作用
Kernel PCA 使用核函数处理非线性降维问题
Incremental PCA 适合大规模数据，可以分批训练
Sparse PCA 得到稀疏主成分，提高可解释性
Robust PCA 增强对异常值和噪声的鲁棒性

11. PCA 的典型应用

PCA 常见应用场景包括：

  1. 图像压缩

    用较少的主成分表示图像，减少存储空间。

  2. 人脸识别

    经典方法 Eigenfaces 就使用了 PCA 思想。

  3. 数据可视化

    将高维数据降到二维或三维，方便观察数据分布。

  4. 机器学习预处理

    在分类或回归前降低特征数量，提高训练效率。

  5. 异常检测

    低方差主成分中有时会包含异常信息。

  6. 推荐系统

    对用户-物品矩阵进行降维，提取潜在特征。

12. 一个常见误区

很多人会误以为：

PCA 就是从原始特征中挑几个重要特征，丢掉其他特征。

这是错误的。

PCA 不是“挑选原始特征”，而是“重新组合原始特征”。

例如原来有 100 个特征，PCA 降到 20 维，并不是选出 20 个原始特征，而是生成 20 个新的主成分。

这 20 个主成分，每一个都可能包含原来 100 个特征的信息。

14. PCA 的一句话总结

PCA 的本质是：

找到数据中方差最大的几个正交方向，并将原始数据投影到这些方向上，从而用更少的维度保留尽可能多的信息。

更通俗地说：

PCA 不是扔掉特征，而是把原始特征重新组合，提炼出更少、更核心的“超级特征”。

15. 极简流程总结

PCA 可以概括为四步：

  1. 标准化数据

    让所有特征处在同一个尺度上。

  2. 计算协方差矩阵

    找出特征之间的相关关系。

  3. 求特征值和特征向量

    找到数据方差最大的方向。

  4. 选择前 k 个主成分并投影

    得到低维表示。

最终效果就是：

用更少的特征描述数据，同时尽可能保留原始数据中的主要信息。
