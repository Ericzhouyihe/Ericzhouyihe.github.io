Bagging 集成学习：让多个"笨"模型组成"聪明"团队

什么是 Bagging？

想象你要做一个重要决定，比如买哪只股票。一个人的判断可能有偏差，但如果你问 10 个朋友，综合他们的意见，结果往往更可靠。Bagging（Bootstrap Aggregating）就是这个道理——让多个模型各自学习，最后"投票"得出答案。

Bagging 是集成学习的经典方法之一，核心思想是：

  • 多样性：每个模型看到的训练数据略有不同
  • 协作：多个"弱"模型组合成一个"强"模型
  • 降低过拟合：单个模型可能记住噪声，但多数模型的平均意见更稳健

为什么 Bagging 有效？

方差-偏差权衡

机器学习的误差可以分解为：

Error=Bias2+Variance+Noise\text{Error} = \text{Bias}^2 + \text{Variance} + \text{Noise} Error=Bias2+Variance+Noise

  • 偏差（Bias）：模型预测的平均值与真实值的差距
  • 方差（Variance）：模型对训练数据的敏感程度

单个决策树桩偏差高（太简单），但 Bagging 通过平均降低了方差：

Var(1M∑i=1MXi)=1MVar(X)\text{Var}\left(\frac{1}{M}\sum_{i=1}^{M}X_i\right) = \frac{1}{M}\text{Var}(X) Var(M1​i=1∑M​Xi​)=M1​Var(X)

当 MMM 个模型相互独立时，方差降低为原来的 1M\frac{1}{M}M1​！

数学直觉

假设 3 个模型的预测分别是 90、100、110，真实值是 100：

  • 单个模型可能偏离较大（±10）
  • 平均值 (90+100+110)/3=100(90+100+110)/3 = 100(90+100+110)/3=100 恰好命中！

这就是"三个臭皮匠，顶个诸葛亮"的数学原理。

代码逻辑解析

有放回抽样的实现

  • frac=1.0：抽取与原数据集相同数量的样本
  • replace=True：每次抽样后放回，允许重复
  • 每次抽样约有 63.2% 的原始样本被抽到（至少一次）

为什么不需要去重？

代码中注释掉了 drop_duplicates()，因为：

  • 重复样本是特性而非 bug，它让每个子数据集不同
  • 去重会减少训练样本，降低每个模型的性能

投票机制

np.sign() 函数：

  • 输入 > 0 → 输出 +1
  • 输入 < 0 → 输出 -1
  • 输入 = 0 → 输出 0（实践中很少出现）

应用场景

1. 随机森林

随机森林是 Bagging 的变体：

  • 在每次分裂时随机选择特征子集
  • 进一步增加模型多样性
  • Scikit-learn 中的 RandomForestRegressor 和 RandomForestClassifier

2. 医疗诊断

  • 多个医生（模型）独立诊断
  • 综合意见降低误诊率
  • 适用于高风险决策场景

3. 金融风控

  • 评估贷款违约风险
  • 多个模型从不同角度评估信用
  • 降低单一模型的过拟合风险

4. 图像分类

  • 在 ImageNet 等大型数据集上
  • Bagging 多个 CNN 模型
  • 集成学习常用于竞赛 Top 方案

超参数调优建议

  1. 弱学习器数量（n_trees）
    • 一般 50-500 之间
    • 太少效果不明显，太多计算成本高
    • 观察训练曲线，性能稳定后停止增加
  2. 基学习器复杂度（max_depth）
    • 深度 1-5 的决策树桩常用
    • 太深单个树过拟合，Bagging 效果减弱
    • 太浅欠拟合，集成也无力回天
  3. 抽样比例（frac）
    • 默认 1.0（与原数据集同大小）
    • 可尝试 0.6-0.8 加快训练
    • 比例太小导致每个模型训练不足

总结

Bagging 通过三个简单步骤实现强大的集成效果：

  1. 自助采样：有放回抽样生成多个训练子集
  2. 并行训练：独立训练多个弱学习器
  3. 结果聚合：平均（回归）或投票（分类）

优点：

  • 降低过拟合，提高模型稳定性
  • 并行训练，易于分布式实现
  • 对噪声和异常值鲁棒

局限：

  • 无法降低偏差（如果基学习器太弱）
  • 计算成本是单模型的 MMM 倍
  • 可解释性降低（黑盒模型）

下次遇到模型性能瓶颈时，不妨试试 Bagging——让"团队智慧"为你的模型加分！
